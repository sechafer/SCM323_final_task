01 · Context

The Problem: Digital Clutter as Waste

Kaizen identifies waste (muda) in any form — including digital environments. A disorganized desktop and unnecessary startup processes are textbook examples of "searching waste" and "waiting waste" that compound across every work session.

🗂️

Visual Clutter

Dozens of files scattered on the desktop with no naming convention or grouping made locating items a time-consuming hunt.

⏳

Startup Lag

All non-essential apps were enabled at login, adding unnecessary CPU and memory load from the first second of use.

🔍

Search Waste

Finding the right file required a manual visual scan across the entire screen — every single time a task began.

🔄

Repetition

This friction was experienced multiple times daily, making it the perfect Kaizen candidate: frequent, controllable, measurable.

02 · Before vs After

Identifying & Eliminating Waste

The Kaizen improvement targeted two specific areas: the visual state of the desktop and the Windows Task Manager startup list. Both were analyzed, photographed (before), then improved and documented (after).

❌ Before
  • Desktop covered in unorganized files & shortcuts
  • No folders or logical grouping
  • All startup programs enabled in Task Manager
  • High system load at login
  • Task completion time: ~45 seconds
  • High cognitive load when context-switching
→
✅ After
  • Desktop cleaned — only active project shortcuts remain
  • Files grouped in clearly labeled folders
  • Unnecessary startup apps disabled in Task Manager
  • Faster boot & login experience
  • Task completion time: ~15 seconds
  • Calm, focused visual environment

03 · Kaizen Actions

Step-by-Step Improvement Process

Following the PDSA cycle, each action was planned, executed, measured, and standardized for daily replication — no heroic effort required.

1
Observe

Baseline Measurement

Timed how long a common task (finding & opening a project file) took on the original messy desktop: average 45 seconds. Screenshots captured for documentation (windows1, windows4).

2
Organize

Desktop Declutter

Removed all non-essential icons. Created named folders (Projects, Tools, Archive). Aligned remaining shortcuts in a minimal grid at the left edge. Wallpaper set to a plain dark tone to reduce visual noise.

3
Optimize

Startup Program Audit

Opened Task Manager → Startup Apps. Disabled all programs not needed at login. Kept only core security & essential work tools. Result: boot-to-usable reduced by ~40 %.

4
Measure

Post-Improvement Timing

Re-ran the same task on the improved desktop: 15 seconds. Reduction confirmed. Screenshots captured (windows2, windows3) for the video case study.

5
Standardize

Maintain the Standard

A simple rule: desktop must have ≤ 10 items at end of each work day. Task Manager startup list reviewed weekly. Kaizen is never a one-time event — it requires daily discipline.
